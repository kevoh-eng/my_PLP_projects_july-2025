HTML5 Semantic Structure

Why Use Semantic Tags?

Semantic HTML makes it easier for search engines and screen readers to understand what each part of the page does.

Common Elements

  • <header> – Top section of the page
  • <nav> – Menu or navigation links
  • <main> – Main content area
  • <section> – Groups related content
  • <article> – Standalone content blocks
  • <aside> – Side info like tips or links
  • <footer> – Bottom of the page

Accessibility Basics

Key Principles

Good websites are accessible to everyone. The WCAG principles are:

  1. Perceivable – Content should be visible and understandable
  2. Operable – Can be used with keyboard and mouse
  3. Understandable – Easy to read and navigate
  4. Robust – Works on different browsers and devices

Accessibility Tips

  • Use headings in the correct order (H1, H2, H3...)
  • Add alt text for images (if used)
  • Make sure keyboard users can navigate
  • Use clear labels and simple language
